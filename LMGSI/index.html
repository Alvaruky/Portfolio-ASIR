Enlaces Anclas/Internos

Enlaces dentro de la misma página. Son enlaces que nos permiten posicionarnos en un punto concreto del documento dentro de la misma página. Por lo tanto, creamos un enlace que nmos lleva a un bloque de contenido de la página.

ID Es un atributo presente en cualquier elemento de la página y nos permite identificar dicho elemnto (dengtro de la propia pagina) CUIDADO no se pueden repetir los valores del identificador para ese atributo en cualquier otro elemento es unico.

Nuestros enlaces internos/anclas este identificador será el denominado Ancla.

<h1 id="Ancla"> Ir a mitad </h1>

Antes en lugar de utilizarse el id se utilizaba el atributo name dentro de la propia etiqueta <a>

Dos variantes de enlaces internos:

Es decir, dentro de una carpeta accedo ala mitad o al final de la misma. Indica la posición interna en una URL absoluta. Acceso a la Wikipedia Historia de Palencia.

Atributos de Elementos <a>

El único atributo obligatorio en la etiqueta <a> es el href que me permite indicar el destino del enlace. Pueden existir algunos atributos más que se pueden utilizar en la etiqueta <a>.

- El atributo tarjet Permite indicar como se muestra la página de destino. tarjet="_blank" nos permite abrir el enlace a una página nueva, ya que esta es la opció más utilizada.
- El atributo download Permite que al hacer clic en el enlace, se abra automáticamente una pestaña y se descargará el fichero. Claro está de que ha este atributo se le puede añadir un nombre nuevo:
download="prevencion.pdf" lo cual nos va a descargar un fichero con el nombre que se ha indicado en el download.
NOTA El archivo original que se va a descargar se tiene que enyectar en el servidor. href="documentos/fichaprev.pdf"

- El atributo type Simplemente nos permite indicar el tipo de contenido. A continuación puedes observar varios ejemplos.
<a href="documentos/fichaprev.pdf">
type="application/pdf" En este ejemplo nos indica que el enlace hace referencia a un documento de tipo pdf.
<a href="img/bacalao.png" type="image/png"> descargar imagen </a>

Introducción a las Imágenes

Las imágenes son fundamentales para crear páginas web atractivas al usuario. Son el primer contenido NO TEXTUAL que se planteó incorporar en el estándar HTML.

Tipos:

Formato .jpg: Imágenes que ocupan muy poco espacio, alta comprensión. NO ADMITEN ANIMACIONES NI TRANSPARENCIAS. Formato habitual.

Formato .gif: Imágenes con hasta 256 colores son buenas para dibujos y se pueden utilizar para animaciones (su uso habitual).

Formato .png: Imágenes fotográficas comprimidas al estilo jpg pero con más calidad. Permite utilizar canales de opacidad (transparencias), permiten utilizar un fondo y editar su opacidad.

Formato .webp: Propio de Google, para hacer la competencia a .png y .jpg ocupan menos y tienen menor perdida de calidad. El problema está en que solo es soportado por Google y por Opera.

Formato .SVG: Formato vectorial de imágenes. Tienen la ventaja de que nunca pierden calidad independientemente de cuanto ampliemos la foto. Problemas de compatibilidad y de tamaño (pesan mucho).


Elementos:

Elemento Figure: Agrupa elementos relativos a una imágen como la propia imágen, el título, el pie...

Elemto Figcaption: Permite indicar el título de una imagen, dentro del elememto anterior (figure) haciendo efecto de un título con pie de imágen.

Insercción de Imágenes

Las imágenes se colocan con la etiqueta <img>, importante de todo esto es que es una etiqueta que no tiene cierre.
Se considera texto, en el sentido de que yo puedo añadir otras imágenes en la misma linea e incluso en el propio texto "inline". Para colocar de forma adecuada las imágenes se consigue mediante maquetación de CSS.

Atributo:

Alt: Descripción de la foto y es un caracter obligatorio que indica un texto alternativo a la imágen. Este texto aparece cuando la imágen no puede cargarse. Este texto se tiene en cuenta en los buscadores con el fin de identificar la imágen y a parte de eso permite conocer que tiene la fotografia (ALT).

Src: Atributo obligatorio, añadimos la ruta de a imágen.

Width / Height: El primero me permite indicar la anchura de la imágen y el segundo me permite indicar la altura de la imágen.
NOTA: Si añadimos tan solo uno de los dos atributos es capaz de calcular el otro valor a partir del añadido. widht="300px";

Etiqueta Picture junto con Source

Permite mostrar varios medios en una misma página web. Si source va dentro de la etiqueta picture puede utilizarse para imágenes y esta tiene tres atributos:

Srcest: Indicar la dirección del archivo de la imágen que queremos mostrar.

Type: Me permite indicar el MIME del recurso que vamos a mostrar.

MIME: Hace referencia al tipo de información que queremos mostrar.

Media: Permite consultar el tipo de imágen que se esta mostrando en la página o para el tamaño de la página web si queremos añadir diferentes páginas con distintos tamaños.
Ejemplo: Inicialmente va a intentar mostrar la imágen del primer source, si el navegador no es capaz de interpretar el tipo de esa imágen mostrará el segundo source (si existe) y si no mostrará el valor de la etiqueta imágen, y si no encuentra la etiqueta imágen mostrará el texto del atributo alt.

<picture>
<source media="(min-with:1000px)" srcest="../../img/aceituna.jpg">
<source srcest="../../img/cebolla.jpg" media="(min-with:700px)">
<img srcest="../../img/patata.jpg" alt="alimentación">
</picture>

Elementos Multimedia

Esencialmente la base es la inclusión de imágenes pero cada vez más se agregan a las páginas web vídeos o sonidos en definitiva en elementos visuales que hagan a las páginas más atractivas.
El vídeo es un tipo de contenido que ocupa mucho espacio por lo que siempre se intenta comprimir al máximo su tamaño para ello se utliza software especial por estos motivos.


Formatos de vídeo utilizados en html5;

MP4: Probablemente uno de los más populares ya que es compatible con los navegadores (tanto de pcs como de móviles).

WebM: Creado por Google pero es compatible CASI con todos los ordenadores, comprime mejor que el MP4 lo que hace que sea más optimo a la hora de visualizar el vídeo...

OGG VIDEO:Es un formato abierto y sin patentes, vamos que es de codigo abierto.


Formas de incluir un vídeo:

< a href="../../img/video.mp4"> Ver video < /a >
< video src="../../img/video.mp4"> controls < /a >
< video src="../../img/video.mp4"> controls poster="../../img/patata.jp" < /a >
